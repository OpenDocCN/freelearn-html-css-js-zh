Preface

The field of web development is an exciting one, where rapid technological changes are the order of the day. This can make it a challenging field when it comes to knowing where to start. When I broke into this field, all we really needed was a basic text editor such as Notepad, and some time to look at the code behind websites to glean the basics of what was involved in building them. Being self-taught was still difficult back then, but it was certainly easier as there wasn’t nearly as much to learn.

Things are far from that simple now though! Figuring out where to begin your journey is tricky. And then, when you do figure that out and learn the basics of the technology, you have to confront the challenge of entering the field in a professional capacity, which can be difficult when you don’t have a lot to demonstrate your capabilities.

And then once you figure out how to crack that nut, the challenge shifts to keeping up with the constant churn of technology, continuing to expand your capabilities, and learning how to fit in and do good work.

So yes, while it’s an exciting field, it’s also a challenging one. But I hope that I, through this book, will be able to draw on nearly 30 years of professional experience to help you start the journey on solid footing, building up a good foundation in technology, plus a lot of other skills you’ll need to be a successful web developer.

Who this book is for

If you’re an aspiring web developer with a passion for problem-solving and the thrill of crafting digital innovations, then this book is for you. Whether you’re a dedicated learner, a tech enthusiast, or someone interested in starting a career in this field, this book offers valuable insights.

Designed to be accessible to individuals without any prior technical or software development knowledge, this book provides a high-level overview and a foundational introduction to the world of web development. It’s the perfect starting point for those looking to step into a vast and exhilarating realm.

What this book covers

In Chapter 1, So, You Want to Be a Web Developer, eh?, you will learn what it means to be a web developer and what the job entails, both technically and on a personal level.

In Chapter 2, Bridging the Divide – Client (Frontend) versus Server (Backend) Development, you will start to learn the difference between the frontend and backend, how they work together, and the technological underpinnings of both.

In Chapter 3, Expanding the Foundations – Building a User Registration Page, Front to Back!, you will continue to be exposed to the foundations of web development and review a complete – albeit simple – website that demonstrates all the key components.

In Chapter 4, Managing, Securing, and Working with Code, you will begin to get a feel for the types of tools that web developers use and begin to understand how the code you write is built, managed, and deployed.

In Chapter 5, Understanding the User Experience, Deployment Options, and DevOps, you will go a bit deeper into getting the work you produce into a state that will work well for users and how you can actually make it available to them.

In Chapter 6, Exploring Modern Frontend Development, you will look at some of the current state-of-the-art frameworks, libraries, and toolkits that are used to build the frontend of websites and web apps.

In Chapter 7, From Front to Back – More on Backend Tools and Technologies, you will take a deeper dive into the backend technologies used to build and run the server-side code; this is the code that your frontend code makes requests to in order to get its work done.

In Chapter 8, Writing Less Code – CMSs, No-Code Solutions, and Generative AI, you will learn about the tools that allow web developers to build solutions without writing code, or at least writing less of it.

In Chapter 9, Landing Your First Job, you will begin to look at non-technical topics – for the most part – that have to do with getting you in the door and finding your first web development job.

In Chapter 10, Discovering the Keys to Quality Work as a Web Developer, you will see the techniques, habits, and tools that good web developers use to ensure they deliver quality work each step of the way.

In Chapter 11, Reviewing Soft Skills (They Make You Hard to Deny), you will learn about skills that are generally non-technical but that any good web developer needs, including things such as interpersonal relationship skills and communication skills.

In Chapter 12, Growing Your Career, you will get a feel for the importance of continued learning and growth in the field, since the technologies and demands are constantly changing, so you must do the same to be successful!

Conventions used

There are a number of text conventions used throughout this book.

Code in text: Indicates code words in text, database table names, folder names, filenames, file extensions, pathnames, dummy URLs, user input, and Twitter handles. Here is an example: “The HTML document itself is created by starting with an <html> tag and ending with its closing partner </html>.”

A block of code is set as follows:


<h1>Luke Skywalker</h1>
<div id="relatives">Anakin, Padme, Leia</div>
<p class="friends">Obi-Wan, Han, Chewie</p>

When we wish to draw your attention to a particular part of a code block, the relevant lines or items are set in bold:


<h1>Luke Skywalker</h1>
<div id="relatives">Anakin, Padme, Leia</div>
<p class="friends">Obi-Wan, Han, Chewie</p>

Any command-line input or output is written as follows:


npm init

Bold: Indicates a new term, an important word, or words that you see onscreen. For instance, words in menus or dialog boxes appear in bold. Here is an example: “Now, to set up a virtual machine in VirtualBox, click the Machine menu, then the New button.”

Part 1:Introducing the Basics of Web Development

In this part, you will begin to learn the basics of what it means to be a web developer. We’ll talk about what a web developer is and what a typical workday for one might look like, plus the job titles and types of work that define it.

We’ll also start exploring the technology involved, including the “big three” of HTML, CSS, and JavaScript. You’ll gain an understanding of the client/server divide, what the frontend and backend are, and how they work together. You’ll gain some knowledge on networking and the types of websites and applications a web developer could be asked to build.

Finally, you’ll get your first experience building an actual website to see how all the pieces fit together to form a final product.

This part includes the following chapters:

  • Chapter 1, So, You Want to Be a Web Developer, eh?
  • Chapter 2, Bridging the Divide – Client (Frontend) versus Server (Backend) Development
  • Chapter 3, Expanding the Foundations – Building a User Registration Page, Front to Back!
